<p class="MsoNormal" style="margin-bottom:6.0pt"><b style="mso-bidi-font-weight:
normal"><span style='font-size:16.0pt;font-family:"Eras Medium ITC";color:#99CC00'>BuddyChat
and StructuredChat 4.0<o:p></o:p></span></b></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:#FF6600'>3.1.<span style="mso-spacerun:yes">  </span>Introduction<o:p></o:p></span></b></p>
<p class="MsoNormal" style="text-indent:.5in"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>In version 2,
buddy group was its own capability modified from the chat capability included
with ConferenceXP, which allowed users to choose a group of buddies that they
wished to converse with.<span style="mso-spacerun:yes">  </span>In CXP+I-MINDS
version 3, buddy group became a capability mode of merged capability.<span style="mso-spacerun:yes">  </span>Along with the additions of advanced chat
functionality it included a buddy manager from which students could alter who
is and who is not included in their buddy group.<span style="mso-spacerun:yes">  </span>In version 4, the options menu from which to
access ‘Display Statistics’, ‘Database Search’ and ‘Manage Buddy’ were moved to
the main CXP+I-MINDS form.<span style="mso-spacerun:yes">  </span>Also, with
the addition of moderator assigned structured groups, a new communication mode
was added named Structured Chat. The functionality is nearly identical, with
the exceptions that users cannot change their structured group members, and if
their structured group is changed by the administrator, the user will receive
the current topics being discussed in the new group they enter.<span style="mso-spacerun:yes">  </span>Because of their marked similarities, we will
only describe the functionality of Structured Group Chat, although the
description applies to both.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:#FF6600'>3.2.<span style="mso-spacerun:yes">  </span>Flow of Control<o:p></o:p></span></b></p>
<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><span style='font-family:"Eras Medium ITC"'><span style="mso-spacerun:yes">   
</span><o:p></o:p></span></p>
<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><span style='font-family:"Eras Medium ITC"'>Unlike the question answering mode in
merged capability, there is only one version of each of the chat modes.<span style="mso-spacerun:yes">  </span>These modes are designed to be used by both
teachers and students simultaneously.<span style="mso-spacerun:yes"> 
</span>Participants can send messages to each other by using these modes.<span style="mso-spacerun:yes">  </span>It is important to mention that only
participants running the chat modes in merged capability who are in the
sender’s classroom will receive the message and only participants in the
student’s group will display it. <o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><span style='font-family:"Eras Medium ITC"'>In a Chat capability, it is possible to
select a previously received message by just clicking on it in the message text
box.<span style="mso-spacerun:yes">  </span>The selected message is added to
the response text box.<span style="mso-spacerun:yes">  </span>While the
response text box contains a message, any new messages sent by the user will be
'in response' to this message. To clear the message just right-click the
response text box to bring up the menu and select 'Clear'.<span style="mso-spacerun:yes">  </span>To view the Message Tree for the previously
received message just select 'History' from popup menu.<span style="mso-spacerun:yes">  </span>This will cause the MessageTree form to
become visible, displaying the tree view for the message tree (Figure 3.3).<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><span style='font-family:"Eras Medium ITC"'>For Chat capabilities we introduce the
concept of Message Trees.<span style="mso-spacerun:yes">  </span>Message Trees
provide the ability for one message to initiate an expanding flow of
dialog.<span style="mso-spacerun:yes">  </span>A Message Tree starts with a
single topic started by one of the students (Figure 3.2).<span style="mso-spacerun:yes">  </span>This topic forms the root of the Message
Tree.<span style="mso-spacerun:yes">  </span>Topics with unseen messages are
displayed in bold in the topics field of the form (Figure 3.1).<span style="mso-spacerun:yes">  </span>When another student responds to a topic,
this linkage is stored on the CXP+I-MINDS database. New messages can respond to
either the original topic or any previous response.<span style="mso-spacerun:yes">  </span>The tree of a conversation can be viewed by
right clicking in the Respond to: panel and selecting History.<span style="mso-spacerun:yes">  </span>By using a recursive algorithm on the
CXP+I-MINDS database the stored messages are extracted and added to a tree view
data object to facilitate display.<o:p></o:p></span></p>

<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";
color:#FF6600'>3.3.<span style="mso-spacerun:yes">  </span>Intelligent Features<o:p></o:p></span></b></p>
<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><span style='font-family:"Eras Medium ITC"'>In order to better support users, we have
begun adding intelligent functionality to both BuddyChat and
StructuredChat.<span style="mso-spacerun:yes">  </span>To accomplish this, the
user agents monitor each topic the user is involved in.<span style="mso-spacerun:yes">  </span>If the user has many unread messages in a topic,
the agent notices this condition and notifies the user, asking them if they
would like to view the unread messages.<span style="mso-spacerun:yes"> 
</span>If a user’s contribution level is low in a topic, the agent encourages
the user to increase their participation, and offers to display the topic if it
is not the one currently focused on.<span style="mso-spacerun:yes"> 
</span>Also, in BuddyChat, if the agent sees that a nonbuddy is contributing
heavily to a topic started by a buddy and the user has extra room to add new
buddies, the agent recommends adding the nonbuddy to the user’s buddy
list.<span style="mso-spacerun:yes">  </span>However, we know that not all
users will want to be bothered by such messages, so we added reinforcement
learning to the agents so they will stop notifying the user of these different
conditions based on the user’s reaction to the notifications.<span style="mso-spacerun:yes">  </span>Whenever the agent notifies the user, it
remembers the user’s response and is increasingly less likely to notify the
user again if the user chooses to ignore the agent’s alert.<o:p></o:p></span></p>
<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><span style='font-family:"Eras Medium ITC"'>For users and moderators who join
sessions that have already begun or are dropped because of a bad network
connection and rejoin, we have also added functionality to allow the agents to
request any existing messages from other participants.<span style="mso-spacerun:yes">  </span>For BuddyChat, the user agent asks the agents
of the user’s buddies for all messages in topics either the buddy or they
themselves started.<span style="mso-spacerun:yes">  </span>The agents then
respond with such messages, and the user’s agent rebuilds its collections and
displays all existing topics.<span style="mso-spacerun:yes">  </span>Similarly,
for StructuredChat, the agents request all messages from group members and
display those as well.<span style="mso-spacerun:yes">  </span>Because
moderators can chat with anyone, user agents also ask for topics created by
moderators, and moderators ask for topics from everyone.<span style="mso-spacerun:yes">  </span>Finally, when moderators modify existing
structured groups, the agents also work together to insure that all members of
each group after the change have all of the messages previously sent within the
group.<span style="mso-spacerun:yes">  </span><o:p></o:p></span></p>
<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";
color:#FF6600'>3.4.<span style="mso-spacerun:yes">  </span>Buddy Group Manager<o:p></o:p></span></b></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>Figure 3.4 shows
a screenshot of the buddy manager form.<span style="mso-spacerun:yes"> 
</span>The buddy manager becomes visible whenever the user selected ‘Manage
Buddy Form’ from the main CXP+I-MINDS form.<span style="mso-spacerun:yes"> 
</span>The buddy manager form shows all the participants who are currently
stored in the CXP+I-MINDS database and indicates whether they are on or
offline.<span style="mso-spacerun:yes">  </span>There are two checkboxes for
each participant, one for adding the participant as a buddy and the other for
adding her as a temporary buddy.<span style="mso-spacerun:yes">  </span>Temp
buddies are counted as buddies for purposes of receiving messages, but the
relationship is asymmetric rather than symmetric.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>Maintaining the
symmetric relationship between buddies requires a handshaking procedure between
the two participants and the CXP+I-MINDS database.<span style="mso-spacerun:yes">  </span>Because of the requirement of symmetric buddy
relationships, you can only add a buddy if that participant is currently
online.<span style="mso-spacerun:yes">  </span>When you check the Buddy box for
a participant, your agent sends a message to your potential buddy.<span style="mso-spacerun:yes">  </span>When the message is received, her agent will
popup a confirmation box asking her if she would like to add you as a
buddy.<span style="mso-spacerun:yes">  </span>Based on her decision, her agent
sends one of the following three responses: (1) she has added you as a buddy,
(2) she has declined your offer of being a buddy, or (3) she cannot become your
buddy because her buddy group is full.<span style="mso-spacerun:yes"> 
</span>Because removing a buddy does not require such handshaking, students are
able to remove a buddy whether they are online or offline. When that student
logs in next, the buddy manager will simply show the two are no longer
buddies.<span style="mso-spacerun:yes">  </span><o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>

<p class="MsoNormal" style="mso-layout-grid-align:none;text-autospace:none"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";
color:#FF6600'>3.5.<span style="mso-spacerun:yes">  </span>Structured Group

<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>To facilitate
the formation of structured groups, a new component has been added to version 4
called Structured Group Manager.<span style="mso-spacerun:yes">  </span>This
powerful utility allows moderators to create sets of Structured Groups which
can then be used in collaborative quizzes and structured chat.<span style="mso-spacerun:yes">  </span>When the groups are set up, a moderator can
save those groups as a set of groups. This allows the moderator to keep a list
of possible group combinations to assign for a particular assignment or time
period. <o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>Group assignment
begins with the moderator creating a new set described above by entering a name
in the new set field then clicking ‘Create new Set’ (Figure 3.5).<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p align="center" class="MsoNormal" style="text-align:center"><span style='font-family:"Eras Medium ITC"'><!--[if gte vml 1]><v:shape id="_x0000_i1029"

<p class="MsoNormal" style="margin-left:.5in;text-indent:-.25in;mso-list:l0 level1 lfo5"><span style="font-family:Symbol;mso-fareast-font-family:Symbol;mso-bidi-font-family:

<p class="MsoNormal" style="margin-left:.5in;text-indent:-.25in;mso-list:l0 level1 lfo5"><span style="font-family:Symbol;mso-fareast-font-family:Symbol;mso-bidi-font-family:

group in the ‘Group B’ drop down menu, checking which attribute you would like

Group B’. <o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:.5in;text-indent:-.25in;mso-list:l0 level1 lfo5"><span style="font-family:Symbol;mso-fareast-font-family:Symbol;mso-bidi-font-family:

</span></span></span><span style='font-family:"Eras Medium ITC"'>To

‘Delete Group’, which places all members of the group to the unassigned
students category, and removes this group from the set.<span style="mso-spacerun:yes">  </span><o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:.5in;text-indent:-.25in;mso-list:l0 level1 lfo5"><span style="font-family:Symbol;mso-fareast-font-family:Symbol;mso-bidi-font-family:
Symbol"><span style="mso-list:Ignore">·<span style='font:7.0pt "Times New Roman"'>       
</span></span></span><span style='font-family:"Eras Medium ITC"'>Members
can be moved between groups by selecting the two groups involved in the drop
down menus, highlighting the participants to be moved, and then clicking the
arrow indicating the direction to be moved. Not only can you move students
between groups but also from the Unassigned Participants category into a new
group. <o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:.5in;text-indent:-.25in;mso-list:l0 level1 lfo5"><span style="font-family:Symbol;mso-fareast-font-family:Symbol;mso-bidi-font-family:
Symbol"><span style="mso-list:Ignore">·<span style='font:7.0pt "Times New Roman"'>       
</span></span></span><span style='font-family:"Eras Medium ITC"'>Another
option for group formation is filling the group with randomly selected
participants from the unassigned list. To do this, a group is selected from the
‘Group B’ combo box, then click ‘Random Fill Group B’ which adds random members
from the unassigned list up to the size of the group.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>At each step,
the moderator has the option of saving the changes to the set, or reverting the
set to the last saved state. <o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>The Set Manager
tab has many convenient tools to manage structured groups.<span style="mso-spacerun:yes">  </span>Any saved set can be selected in the set drop
down box in the top left of the window.<span style="mso-spacerun:yes"> 
</span>A list of the groups in that set with the members of each group is then
displayed. The selected set can also be renamed by entering the new name in the
rename set text box, then clicking ‘Rename Set’.<span style="mso-spacerun:yes">  </span>Groups can also be created and assigned in
one fell swoop by specifying the number of groups desired in the Auto Set text
box, then clicking ‘Random Auto Set’. This creates that number of even sized
groups, and randomly places all of the students into the groups evenly. This
allows for fast group set creation and assignment. Finally when the editing on
the set is completed, click the ‘Save All, and Make Active Set’. This will save
all changes and then make the selected Set the ‘Active’ set, or the set used in
collaborative assignments and Structured Group Chat.<o:p></o:p></span></p>
