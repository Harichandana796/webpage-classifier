<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>We have
significantly revised our <span class="SpellE">QuestionAnswering</span> and <span class="SpellE">BuddyGroup</span> capabilities.<span style="mso-spacerun:yes"> 
</span>Further, we have merged both the buddy group and question answering
capabilities into one capability as an alternative.<span style="mso-spacerun:yes">  </span><o:p></o:p></span></p>

<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>There were two
good reasons for having this merged capability.<span style="mso-spacerun:yes"> 
</span>The first is based on the underlying functionality of capabilities in <span class="SpellE">ConferenceXP</span>.<span style="mso-spacerun:yes">  </span>Each
capability instance was designed to run autonomously from any other
capabilities being run on the same machine.<span style="mso-spacerun:yes"> 
</span><span class="SpellE">ConferenceXP</span> provides all the networking
between the same <span class="GramE">capability</span> being run on multiple
machines, but provides no way to communicate between multiple instances or
different capabilities running on the same machine.<span style="mso-spacerun:yes">  </span>This created a good deal of work for us, as
we had to make sure that one or more machines running one or both of our
existing capabilities created no concurrency issues with the database.<span style="mso-spacerun:yes">  </span>The second reason was scalability; suppose we
want to add a new I-MINDS capability or add new tables to the I-MINDS
database.<span style="mso-spacerun:yes">  </span>In the first case we have to
extensively test all three capabilities working in tandem to make sure no
concurrency issues occurred.<span style="mso-spacerun:yes">  </span>In the
second case, we would have to re-factor all the database queries separately for
each capability.<span style="mso-spacerun:yes">  </span>This was a time
intensive process and had the potential for introducing new bugs into one or
more I-MINDS capabilities.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>Here is a
synopsis of our changes:<o:p></o:p></span></p>

<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>As with previous
versions, the question answering mode in version 3 retains both the student and
teacher version.<span style="mso-spacerun:yes">  </span>Both versions were
originally based on the chat capability that comes with <span class="SpellE">ConferenceXP</span>.<span style="mso-spacerun:yes">  </span>The teacher version was heavily modified to
enable lists of previously received questions to be displayed and sorted.<span style="mso-spacerun:yes">  </span>Based on feedback from a test run by <st1:place w:st="on"><st1:placename w:st="on">Bellevue</st1:placename> <st1:placetype w:st="on">University</st1:placetype></st1:place> (www.bellevue.edu) during the
summer of 2006 we have modified the teacher form to make it more user-friendly.<span style="mso-spacerun:yes">  </span>The first major change in the teacher for is
the inclusion of tabs to partition the messages into received, answered and
discarded.<span style="mso-spacerun:yes">  </span>We also added the ability to
reset an answered or discarded message.<span style="mso-spacerun:yes"> 
</span>The second major change was to allow teachers to send messages that were
not responses.<span style="mso-spacerun:yes">  </span>This feature is described
in more detail below.<span style="mso-spacerun:yes">  </span>Last but not
least, teacher can now broadcast responses to all students, instead of just the
one who sent the question.<span style="mso-spacerun:yes">  </span>Both versions
have been modified to use the new message objects rather than the XML
tags.<span style="mso-spacerun:yes">  </span>Other than this, the student
version remains unchanged in version 3.<span style="mso-spacerun:yes">  </span><o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span class="GramE"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:blue'>Teacher Flow of Control.</span></b></span><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC"'><span style="mso-spacerun:yes">  </span></span></b><span style='font-family:"Eras Medium ITC"'>The
teacher form displays all the questions that have been previously asked by
students in the classroom.<span style="mso-spacerun:yes">  </span>Questions
that have already been answered or discarded will appear in the appropriate
tabs.<span style="mso-spacerun:yes">  </span>Questions that are still waiting
for a response will appear in the ‘Received Questions’ tab, sorted by
score.<span style="mso-spacerun:yes">  </span>When the teacher right-clicks on
a message in the ‘Received Questions’ table he or she has the option of either
answering or discarding the question.<span style="mso-spacerun:yes">  </span>If
the teacher selects the question, it will appear in the ‘Current Question’ box.<span style="mso-spacerun:yes">  </span>The teacher can then write a response to this
question and press ‘Answer’ button.<span style="mso-spacerun:yes">  </span>A
new message with the <span class="SpellE">QA_Response</span> message type is then
created with the response and the message object is then multicast using the <span class="SpellE">SendObject</span> method to the rest of the classroom.<span style="mso-spacerun:yes">  </span>When the teacher answers or discards the
question, the question answering agent will modify the weights of the keywords
it contains.<span style="mso-spacerun:yes">  </span>See the next section for
more details. <b style="mso-bidi-font-weight:normal"><o:p></o:p></b></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>In version 3, we
have added the ability for the teacher to send message to the students rather
than just waiting to respond to their questions.<span style="mso-spacerun:yes">  </span>We do this by maintaining a list of students
who are currently in the classroom and running the Q&amp;A mode.<span style="mso-spacerun:yes">  </span>By selecting individual students or selecting
the ‘To All’ checkbox the teacher can determine which students will receive the
message.<span style="mso-spacerun:yes">  </span>When the teacher presses the
‘Message’ button, whatever she has written is encapsulated into a message
object with the <span class="SpellE">QA_Message</span> message type and sent to
all the students who are selected.<span style="mso-spacerun:yes">  </span>The
students can easily distinguish between messages and responses from the teacher
because the Q&amp;A agent indicates this on the student form.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>As mentioned
above, the teacher version listens for questions asked by students.<span style="mso-spacerun:yes">  </span>When such a question is received the question
answering agent first determines what score the question should receive.<span style="mso-spacerun:yes">  </span>This is done by first parsing and tokenizing
each word in the question.<span style="mso-spacerun:yes">  </span>For more
information please consult the Question Answering Agent section below.<span style="mso-spacerun:yes">  </span>The question immediately appears in the
‘Received Questions’ table and is available to the teacher.<span style="mso-spacerun:yes">  </span>As this table is sorted by score, good
questions will appear higher on the table while bad questions might not even
appear on the table unless the teacher scrolls down.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span class="GramE"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:blue'>Intelligent Agent.</span></b></span><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC"'><span style="mso-spacerun:yes">  </span></span></b><span style='font-family:"Eras Medium ITC"'>The
teacher version of the question answering mode contains an intelligent agent
whose purpose is to aid the teacher in deciding which questions to answer.<span style="mso-spacerun:yes">  </span>It does this by assigning a score to each
question as it is received from the student.<span style="mso-spacerun:yes"> 
</span>This score is determined by the length of the question and the weights
of the keywords it contains.<span style="mso-spacerun:yes">  </span>When a
question is first received it is tokenized and stemmed by a natural language
processing program.<span style="mso-spacerun:yes">  </span>The stems are
checked against the keywords stored in the database.<span style="mso-spacerun:yes">  </span>If a match is found the weight of the keyword
is added to a running total.<span style="mso-spacerun:yes">  </span>This total
is then normalized along with the length of the question to determine the
score.<b style="mso-bidi-font-weight:normal"><o:p></o:p></b></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>In pervious
versions the machine learning weights were <span class="SpellE">hardcoded</span>
for the intelligent agent.<span style="mso-spacerun:yes">  </span>There was
also only one set of keyword weights, hardly suitable for a system that should
support multiple teachers and classrooms.<span style="mso-spacerun:yes"> 
</span>In version 3, we have added the machine learning weights to the database
so they can be adjusted without recompiling.<span style="mso-spacerun:yes"> 
</span>Each classroom has its own set of machine learning weights that the
teacher can modify and the keyword weights are now specific to each teacher and
classroom combination.<span style="mso-spacerun:yes">  </span>This allows <span class="GramE">the I</span>-MINDS system to store the same keyword with different
weights for two different classes taught by the same teacher.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span class="GramE"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:blue'>Adding Keywords.</span></b></span><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC"'><span style="mso-spacerun:yes">  </span></span></b><span style='font-family:"Eras Medium ITC"'>The
teacher can at any time select the ‘Add Keyword’ item from the options
menu.<span style="mso-spacerun:yes">  </span>This brings up a form containing
all the keywords and weights stored in the database for that teacher and
classroom combination.<span style="mso-spacerun:yes">  </span>Unlike in previous
versions, keywords can no longer be deleted by using this form.<span style="mso-spacerun:yes">  </span>New keywords can be added by clicking on the
star row at the bottom.<span style="mso-spacerun:yes">  </span>Existing
keywords can be updated by selecting their entries and typing in a new
weight.<span style="mso-spacerun:yes">  </span>To save the changes simply press
the Update button.<span style="mso-spacerun:yes">  </span>An example of this
form is shown in figure 3.3.<b style="mso-bidi-font-weight:normal"><o:p></o:p></b></span></p>

<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>In version 2,
buddy group was its own capability modified from the chat capability included
with <span class="SpellE">ConferenceXP</span>.<span style="mso-spacerun:yes"> 
</span>In I-MINDS version 3, buddy group is a capability mode of merged
capability.<span style="mso-spacerun:yes">  </span>It retains the same form as
the original capability with text boxes for displaying received messages and
entering new messages.<span style="mso-spacerun:yes">  </span>This mode also
retains the Send button from the original.<span style="mso-spacerun:yes"> 
</span>Improvements to chat include an additional text box where message you
are responding two is displayed.<b style="mso-bidi-font-weight:normal"><span style="mso-spacerun:yes">  </span></b>This text box includes a popup menu with
two items: ‘History’ and ‘Clear’, whose functionality is described in the next<b style="mso-bidi-font-weight:normal"> </b>section. This capability also contains
an ‘Options’ menu with three items: ‘Display Statistics’, ‘Database Search’ and
‘Manage Buddy’.<span style="mso-spacerun:yes">  </span><o:p></o:p></span></p>
