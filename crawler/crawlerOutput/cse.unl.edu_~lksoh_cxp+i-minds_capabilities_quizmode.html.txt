<p class="MsoNormal" style="margin-bottom:6.0pt"><b style="mso-bidi-font-weight:
normal"><span style='font-size:16.0pt;font-family:"Eras Medium ITC";color:#99CC00'>QuizMode<o:p></o:p></span></b></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:#FF6600'>4.1.<span style="mso-spacerun:yes">  </span>Introduction</span></b><span style='font-family:"Eras Medium ITC";color:#FF6600'><o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>Another new mode
introduced in version 4 of CXP+I-MINDS is QuizMode.<span style="mso-spacerun:yes">  </span>This mode is especially useful in educational
settings and allows moderators to create quizzes for users to take.<span style="mso-spacerun:yes">  </span>All quiz content is stored in our database
for automatic retrieval when starting the mode, and all quiz results are
persistently stored as well for future analysis using the QuizStats tool
(described in Section 5.4).<span style="mso-spacerun:yes">  </span>For each
quiz, moderators can also include support material in the form of files for
users to open that might provide more information or help on particular
questions.<span style="mso-spacerun:yes">  </span>Support material are
automatically saved in the CXP+IMINDS directory on both moderator and user
machines to protect against accidental deletion of the original file and to
insure that the files are available until the moderator no longer wants
them.<span style="mso-spacerun:yes">  </span>Please note that due to
ConferenceXP transmission limitations, total support material for a particular
quiz cannot be larger than 750KB in size.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:#FF6600'>4.2.<span style="mso-spacerun:yes">  </span>Creating Quizzes/Questions</span></b><span style='font-family:"Eras Medium ITC";color:#FF6600'><o:p></o:p></span></p>
<p align="center" class="MsoNormal" style="text-align:center"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>To create a new
quiz, moderators must simply load the QuizMode from the main CXP+IMINDS
form.<span style="mso-spacerun:yes">  </span>They will first be created with
the Quiz creation tab shown in Figure 4.1, where moderators can create and edit
existing quizzes by setting the different available properties.<span style="mso-spacerun:yes">  </span>Quizzes have a name and any optional comments
from a moderator, can have a set duration (i.e., the users can no longer choose
answers after the time has expired), and can allow for users to attach their
own comments to any question (e.g., if a user wants to elaborate on why they
chose a particular answer for partial credit).<span style="mso-spacerun:yes"> 
</span><o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>The quiz
creation tab also allows moderators to browse for new support material, view
existing support material, or add support material to the current quiz.<span style="mso-spacerun:yes">  </span>Moderators can also save changes on this tab,
as well as test the quiz they created before sending it to the users.<span style="mso-spacerun:yes">  </span>Finally, this tab also provides functionality
for sending quizzes to users as either individual or collaborative quizzes (to
be explained in Section 4.4), or to view stats from previously assigned
quizzes.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p align="center" class="MsoNormal" style="text-align:center"><span style='font-family:"Eras Medium ITC"'><!--[if gte vml 1]><v:shape id="_x0000_i1026"
 type="#_x0000_t75" style='width:375pt;height:303pt'>
 <v:imagedata src="quizmode_files/image003.png" o:title="Attachment"/>
</v:shape><![endif]--><img border="0" height="404" src="quizmode_files/image004.jpg" v:shapes="_x0000_i1026" width="500"/><o:p></o:p></span></p>
<p align="center" class="MsoNormal" style="text-align:center"><b style="mso-bidi-font-weight:
normal"><span style='font-family:"Eras Medium ITC"'>Figure 4.1: Quiz Creation
Tab</span></b><span style='font-family:"Eras Medium ITC"'><o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>To create
individual questions, as well as to assign them to quizzes, the moderator can
use the Questions tab of QuizMode, shown in Figure 4.2.<span style="mso-spacerun:yes">  </span>To create a new question, the moderator
simply presses the Create New button, which will display a blank question on
the right side of the form.<span style="mso-spacerun:yes">  </span>To change
the name of the question, enter in the question content, as well as enter all
answers, the moderator simply has to choose the appropriate field on the right
side, the type in the formation they want and select the proper answer.<span style="mso-spacerun:yes">  </span>If the moderator wants to provide specific
feedback for when users submit a specific answer (such as why the answer is
correct or incorrect), they can click on the answer they want feedback for,
then type in the feedback in the text box to the left of the Save Question
button.<span style="mso-spacerun:yes">  </span>The properties of the question,
such as whether it has multiple answers, if users can try to answer the
question more than once, and the number of points the question is worth, can
all be set on the left side of the form.<span style="mso-spacerun:yes"> 
</span>Templates are provided for quickly setting properties and include
options such as True/False and Multiple Choice.<span style="mso-spacerun:yes"> 
</span>To save the question, the moderator just needs to press the Save
Question button.<span style="mso-spacerun:yes">  </span>This will add the question
to the Unassigned Questions list if no quiz is selected, else it will assign
the question to the selected quiz.<span style="mso-spacerun:yes"> 
</span>Questions can be moved in and out of quizzes at the top of the left side
of the form, and existing questions can be edited by selected the question,
pressing the edit button, then changing the question content or properties
using the same functionality as creating a quiz.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p align="center" class="MsoNormal" style="text-align:center"><span style='font-family:"Eras Medium ITC"'><!--[if gte vml 1]><v:shape id="_x0000_i1027"

</v:shape><![endif]--><img border="0" height="471" src="quizmode_files/image006.jpg" v:shapes="_x0000_i1027" width="584"/><o:p></o:p></span></p>
<p align="center" class="MsoNormal" style="text-align:center"><b style="mso-bidi-font-weight:
normal"><span style='font-family:"Eras Medium ITC"'>Figure 4.2:<span style="mso-spacerun:yes">  </span>Question Creation Tab<o:p></o:p></span></b></p>
<p align="center" class="MsoNormal" style="text-align:center"><b style="mso-bidi-font-weight:
normal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></b></p>
<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:#FF6600'>4.3. Taking Quizzes</span></b><span style='font-family:"Eras Medium ITC";color:#FF6600'><o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>On the user
side, when taking quizzes, the QuizMode form consists of only the right side of
the moderator form, as shown in Figure 4.3.<span style="mso-spacerun:yes"> 
</span>Notice the quiz timer under the name, which automatically ticks
down.<span style="mso-spacerun:yes">  </span>Once time has run out, users can
no longer answers questions.<span style="mso-spacerun:yes">  </span>If users do
not want to be bothered by the amount of time left, they can click on the
timer, which causes the countdown to disappear.<span style="mso-spacerun:yes"> 
</span>Clicking on the box again causes the timer to reappear.<span style="mso-spacerun:yes">  </span>As previously mentioned, any support material
included with the quiz is available to users taking the quiz.<span style="mso-spacerun:yes">  </span>Users are allowed to advance to the next
question or go back to previous questions.<span style="mso-spacerun:yes"> 
</span>QuizMode tracks the results for each question when an answer is
submitted.<span style="mso-spacerun:yes">  </span>Pressing Done with Quiz ends
the quiz (after confirmation) and sends the results to the moderator. <o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p align="center" class="MsoNormal" style="text-align:center"><span style='font-family:"Eras Medium ITC"'><!--[if gte vml 1]><v:shape id="_x0000_i1028"
 type="#_x0000_t75" style='width:236.25pt;height:369pt'>
 <v:imagedata src="quizmode_files/image007.png" o:title=""/>
</v:shape><![endif]--><img border="0" height="492" src="quizmode_files/image008.jpg" v:shapes="_x0000_i1028" width="315"/><o:p></o:p></span></p>
<p align="center" class="MsoNormal" style="text-align:center"><b style="mso-bidi-font-weight:
normal"><span style='font-family:"Eras Medium ITC"'>Figure 4.3:</span></b><span style='font-family:"Eras Medium ITC"'><span style="mso-spacerun:yes"> 
</span>Quiz Mode User View<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>Quizzes can be
assigned as either individual quizzes or collaborative quizzes.<span style="mso-spacerun:yes">  </span>The former kind allows each user to submit
their own answers for questions without help from anyone.<span style="mso-spacerun:yes">  </span>Collaborative quizzes, on the other hand,
allow students to work in their structured groups and submit a single set of
answers for the whole group.<span style="mso-spacerun:yes">  </span>If the quiz
is taken as a collaborative quiz, control of the form is handled by a
token-based system.<span style="mso-spacerun:yes">  </span>In order to make
changes, a user must request the token.<span style="mso-spacerun:yes"> 
</span>If no one has the token, the moderator agent automatically assigns the
token to the first user to request it.<span style="mso-spacerun:yes"> 
</span>If someone already owns the token, they are prompted if they want to
release the token.<span style="mso-spacerun:yes">  </span>If the owner
confirms, the token is given to the requesting user, else the user’s request is
added to a queue operated by the moderator, and they will be assigned the token
whenever the current owner releases it.<span style="mso-spacerun:yes">  
</span>In order to support collaborative actions, several more options appear
on the quiz form during collaborative quizzes: request token, release token,
and start discussion, which begins a new topic about the question in the
StructuredChat form.<span style="mso-spacerun:yes">  </span><o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>When a user has
the token, they can mark and submit answers, and switch which question is
currently displayed.<span style="mso-spacerun:yes">  </span>Any action taken by
the user with the token is automatically propagated to all of their
collaborative partners’ displays.<span style="mso-spacerun:yes"> 
</span>Whenever the user presses Done with Quiz, the results are submitted to
the moderator for the group.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>In order to
prevent problems caused by users joining sessions late or losing their network
connection, we have added <i style="mso-bidi-font-style:normal">repair</i>
mechanisms to the quiz mode to save completed work.<span style="mso-spacerun:yes">  </span>Whenever a user joins a session, the
moderator agent checks the individual quiz results collections to see if it has
a results set for the joining user.<span style="mso-spacerun:yes">  </span>If
not, the agent sends the first matching quiz found and automatically sends it
to the user.<span style="mso-spacerun:yes">  </span>While the user is taking
the quiz, all of their actions are stored locally in case of loss.<span style="mso-spacerun:yes">  </span>When the user receives a quiz, the user agent
first looks for a corresponding set of stored actions and fills in any work
previously completed by the user.<span style="mso-spacerun:yes">  </span>If the
user had already finished the quiz, but for some reason the results were not
received by the moderator, the user agent automatically sends in the quiz
results without asking the user to take the quiz again.<span style="mso-spacerun:yes">  </span>Whenever the moderator agent receives a new
set of quiz results from a user, it checks the list of assigned quizzes again
to make sure it has received results for every quiz from the submitting
user.<span style="mso-spacerun:yes">  </span>If not, it once again assigns a
make up quiz.<span style="mso-spacerun:yes">  </span>This process continues
until results are received for every quiz.<span style="mso-spacerun:yes"> 
</span>For collaborative quizzes, late joining or dropped users only need to
receive information about quizzes currently being taken, so they automatically
receive information about the state of the current quiz from the moderator,
allowing them to fully participate.<o:p></o:p></span></p>
