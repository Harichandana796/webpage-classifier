<h1 class="post-title">Katie Stolee earned her PhD and is joining Iowa State University</h1>
<span class="post-meta">
								Published by <a href="http://cse.unl.edu/~elbaum/content/?author=2">sebastianelbaum</a> on <a href="http://cse.unl.edu/~elbaum/content/?m=201307">July 8, 2013</a> </span>
</div>
<div class="post-content">
<p>After a distinguished career at UNL, Katherine Stolee defended her dissertation successfully, and will be joining Iowa State University as a faculty member in the Fall 2013. Best of luck Katie!</p>
<p>The abstract of her dissertation on “Solving the Search for Source Code” follows:</p>
<p>Programmers frequently search for source code to reuse using keyword searches. When effective and efficient, a code search can boost programmer productivity, however,<br/>
the search effectiveness depends on the programmer’s ability to specify a query that captures how the desired code may have been implemented. Further, the results often<br/>
include many irrelevant matches that must be filtered manually. More semantic search approaches could address these limitations, yet existing approaches either do not scale, are not  flexible enough to find approximate matches, or require complex specifications.<br/>
We propose a novel approach to semantic search that addresses some of these limitations and is designed for queries that can be described using an example. In<br/>
this approach, programmers write lightweight specifications as inputs and expected output examples for the behavior of desired code. Using these specifications, an SMT<br/>
solver identifies source code from a repository that matches the specifications. The repository is composed of program snippets encoded as constraints that approximate<br/>
the semantics of the code.<br/>
This research contributes the first work toward using SMT solvers to search for existing source code. In this dissertation, we motivate the study of code search and the<br/>
utility of a more semantic approach to code search. We introduce and illustrate the generality of our approach using subsets of three languages, Java, Yahoo! Pipes, and<br/>
SQL. Our approach is implemented in a tool, Satsy, for Yahoo! Pipes and Java. The evaluation covers various aspects of the approach, and the results indicate that this<br/>
approach is effective at finding relevant code. Even with a small repository, our search is competitive with state-of-the-practice syntactic searches when searching for Java<br/>
code. Further, this approach is  flexible and can be used on its own, or in conjunction with a syntactic search. Finally, we show that this approach is adaptable to finding<br/>
approximate matches when exact matches do not exist, and that programmers are capable of composing input/output queries with reasonable speed and accuracy. These<br/>
results are promising and lead to several open research questions that we are only beginning to explore.</p>
