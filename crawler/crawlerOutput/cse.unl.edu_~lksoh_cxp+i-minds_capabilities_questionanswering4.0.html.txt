<p class="MsoNormal" style="margin-bottom:6.0pt"><b style="mso-bidi-font-weight:
normal"><span style='font-size:16.0pt;font-family:"Eras Medium ITC";color:#99CC00'>QuestionAnswering
4.0<o:p></o:p></span></b></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:#FF6600'>2.1.<span style="mso-spacerun:yes">  </span>Introduction<o:p></o:p></span></b></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>As with previous
versions, the question answering mode in version 4 retains both the user and
moderator version.<span style="mso-spacerun:yes">  </span>Both versions were
originally based on the chat capability that comes with ConferenceXP.<span style="mso-spacerun:yes">  </span>In version 3, the moderator version was
heavily modified to enable lists of previously received questions to be
displayed and sorted.<span style="mso-spacerun:yes">  </span>Based on feedback
from a test run by <st1:place w:st="on"><st1:placename w:st="on">Bellevue</st1:placename>
<st1:placetype w:st="on">University</st1:placetype></st1:place>
(www.bellevue.edu) during the summer of 2006, we modified the moderator form to
make it more user friendly.<span style="mso-spacerun:yes">  </span>The first
major change in the moderator for is the inclusion of tabs to partition the
messages into received, answered and discarded.<span style="mso-spacerun:yes"> 
</span>We also added the ability to reset an answered or discarded
message.<span style="mso-spacerun:yes">  </span>The second major change was to
allow moderators to send messages that are not responses.<span style="mso-spacerun:yes">  </span>This feature is described in more detail
below.<span style="mso-spacerun:yes">  </span>Last but not least, moderator can
broadcast responses to all users, instead of just the one who sent the
question.<span style="mso-spacerun:yes">  </span>Screenshots for the User and
the Moderator version of Q&amp;A can be found in Figures 2.1 and 2.2
respectively.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:#FF6600'>2.2.<span style="mso-spacerun:yes">  </span>User Flow of Control<o:p></o:p></span></b></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>Users using the
question answering mode can ask the moderator a question by entering the
question into the box at the bottom of the User form (see Figure 2.1) and
pressing the Send button.<span style="mso-spacerun:yes">  </span>The question
is then encapsulated using a message object with the QA_Question message type
by the user agent.<span style="mso-spacerun:yes">  </span>The resulting message
object is then multicast using the SendObject method to the rest of the
classroom.<span style="mso-spacerun:yes">  </span>In affect, the user version
broadcasts the question to all the other instances of the Question Answering
capability (both user and moderator) running on other machines, but only the
moderator version will be listening for such questions.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>The user version
also listens for responses sent by the moderator version.<span style="mso-spacerun:yes">  </span>Whenever a message with the QA_Response
message type is received, the sender identifier is checked against the
identifier for the local participant.<span style="mso-spacerun:yes">  </span>If
a match is found then the response is being sent to this user version.<span style="mso-spacerun:yes">  </span>The actual response content is displayed
along with the original question in the ‘Recent Answer’ and ‘Recent Question’
boxes.<span style="mso-spacerun:yes">  </span>They are also added to the ‘Prior
Messages’ table.<span style="mso-spacerun:yes">  </span>Figure 2.1 shows how
the user version looks after it has received a response.<span style="mso-spacerun:yes">  </span>User can resend questions to the moderator by
right-clicking on the question or answer and pressing ‘Resend’ on the popup
menu.<span style="mso-spacerun:yes">  </span>Messages sent and broadcast from
moderators are also displayed in the same form.<span style="mso-spacerun:yes">   </span>For these, the message is shown in the
‘Recent Answer’ box.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>

<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:#FF6600'>2.3.<span style="mso-spacerun:yes">  </span>Moderator Flow of Control<o:p></o:p></span></b></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>Moderators using
the QuestionAnswering mode will see the form shown in Figure 2.2.<span style="mso-spacerun:yes">  </span>The moderator form displays all of the
questions that have been previously asked by users in the classroom.<span style="mso-spacerun:yes">  </span>Questions that have already been answered or
discarded will appear in the appropriate tabs.<span style="mso-spacerun:yes"> 
</span>Questions that are still waiting for a response will appear in the
‘Received Questions’ tab, sorted by score.<span style="mso-spacerun:yes"> 
</span>When the moderator right-clicks on a message in the ‘Received Questions’
table he or she has the option of either answering or discarding the
question.<span style="mso-spacerun:yes">  </span>If the moderator selects the
question, it will appear in the ‘Current Question’ box.<span style="mso-spacerun:yes">  </span>The moderator can then write a response to
this question and press ‘Answer’ button.<span style="mso-spacerun:yes"> 
</span>A new message with the QA_Response message type is then created with the
response and the message object is then multicast using the SendObject method
to the rest of the classroom.<span style="mso-spacerun:yes">  </span>When the
moderator answers or discards the question, the agent will modify the weights
of the keywords it contains.<span style="mso-spacerun:yes">  </span>See the
next section for more details. <o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>In version 3, we
added the ability for the moderator to send message to the users rather than
just waiting to respond to their questions.<span style="mso-spacerun:yes"> 
</span>We do this by maintaining a list of users who are currently in the
classroom and using the Q&amp;A mode.<span style="mso-spacerun:yes">  </span>By
selecting individual users or selecting the ‘To All’ checkbox the moderator can
determine which users will receive the message.<span style="mso-spacerun:yes"> 
</span>When the moderator presses the ‘Message’ button, whatever she has
written is encapsulated into a message object with the QA_Message message type
and sent to all the users who are selected.<span style="mso-spacerun:yes"> 
</span>The users can easily distinguish between messages and responses from the
moderator because the agent indicates this on the student form.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>As mentioned
above, the moderator version listens for questions asked by users.<span style="mso-spacerun:yes">  </span>When such a question is received, the agent
first determines what score the question should receive.<span style="mso-spacerun:yes">  </span>This is done by first parsing and tokenizing
each word in the question.<span style="mso-spacerun:yes">  </span>For more
information please consult Section 2.4 on Intelligent Features below.<span style="mso-spacerun:yes">  </span>The question immediately appears in the
‘Received Questions’ table and is available to the moderator.<span style="mso-spacerun:yes">  </span>As this table is sorted by score, good
questions will appear higher on the table while bad questions might not even
appear on the table unless the moderator scrolls down.<o:p></o:p></span></p>
<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></b></p>
<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:#FF6600'>2.4.<span style="mso-spacerun:yes">  </span>Intelligent Features<o:p></o:p></span></b></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>The moderator
version of the QuestionAnswering mode uses the intelligent agent to aid the
moderator in deciding which questions to answer.<span style="mso-spacerun:yes">  </span>It does this by assigning a score to each
question as it is received from the user.<span style="mso-spacerun:yes"> 
</span>This score is determined by the length of the question and the weights
of the keywords it contains.<span style="mso-spacerun:yes">  </span>When a
question is first received it is tokenized and stemmed by a natural language
processing program.<span style="mso-spacerun:yes">  </span>The stems are
checked against the keywords stored in the database.<span style="mso-spacerun:yes">  </span>If a match is found, the weight of the
keyword is added to a running total.<span style="mso-spacerun:yes"> 
</span>This total is then normalized along with the length of the question to
determine the score.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>The intelligent
agent also adjusts the keyword weights up or down.<span style="mso-spacerun:yes">  </span>This is done when the moderator decides to
answer or discard a question.<span style="mso-spacerun:yes">  </span>When the
moderator answers a question the question is once against tokenized and
stemmed.<span style="mso-spacerun:yes">  </span>The weights for keywords found
in the question are increased.<span style="mso-spacerun:yes">  </span>Stems
that are not found in the keywords table are added as keywords with a low
weight.<span style="mso-spacerun:yes">  </span>When the moderator discarded a
question the weights for the keywords found in that question are
decreased.<span style="mso-spacerun:yes">  </span><o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>The natural
language processing program we are using is called “combiner.exe”.<span style="mso-spacerun:yes">  </span>It maintains a taboo list of certain common
words.<span style="mso-spacerun:yes">  </span>All keywords are automatically
normalized by the intelligent agent to fit a certain range.<span style="mso-spacerun:yes">  </span>This prevents the weight associated with a
common keyword from getting so large that it dominates the rest. <o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>In pervious
versions the machine learning weights were hardcoded for the intelligent
agent.<span style="mso-spacerun:yes">  </span>There was also only one set of
keyword weights, hardly suitable for a system that should support multiple
moderators and classrooms.<span style="mso-spacerun:yes">  </span>In version 3,
we added the machine learning weights to the database so they can be adjusted
without recompiling.<span style="mso-spacerun:yes">  </span>Each classroom has
its own set of machine learning weights that the moderator can modify and the
keyword weights are now specific to each moderator and classroom
combination.<span style="mso-spacerun:yes">  </span>This allows the I-MINDS
system to store the same keyword with different weights for two different
classes taught by the same moderator.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-family:"Eras Medium ITC";color:#FF6600'>2.5<span style="mso-spacerun:yes">  </span>Adding Keywords<o:p></o:p></span></b></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>The moderator
can, at any time, select the ‘Add Keyword’ item from the options menu.<span style="mso-spacerun:yes">  </span>This brings up a form containing all the
keywords and weights stored in the database for that moderator and classroom
combination.<span style="mso-spacerun:yes">  </span>Unlike in previous
versions, keywords can no longer be deleted by using this form.<span style="mso-spacerun:yes">  </span>New keywords can be added by clicking on the
star row at the bottom.<span style="mso-spacerun:yes">  </span>Existing
keywords can be updated by selecting their entries and typing in a new
weight.<span style="mso-spacerun:yes">  </span>To save the changes, simply
press the Update button.<span style="mso-spacerun:yes">  </span>An example of
this form is shown in Figure 2.3.<o:p></o:p></span></p>
