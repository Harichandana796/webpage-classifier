<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal" style="margin-bottom:6.0pt"><b style="mso-bidi-font-weight:
normal"><span style='font-size:16.0pt;font-family:"Eras Medium ITC";color:#99CC00'>QuestionAnswering<o:p></o:p></span></b></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>This QuestionAnswering
capability was completed in December 2005.<span style="mso-spacerun:yes"> 
</span>This capability has two views: an instructor application and a student
application.<span style="mso-spacerun:yes">  </span>The person in charge of the
Q&amp;A session (e.g., an instructor) can pick and choose questions to
answer.<span style="mso-spacerun:yes">  </span>The application helps manage the
questions.<span style="mso-spacerun:yes">  </span>The users asking questions
(e.g., a student) can review their questions and answers. They can also ask
questions.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>This capability
also has agent intelligence.<span style="mso-spacerun:yes">  </span>The
instructor application scores each question based on keywords and also “machine
learns” from the instructor about what keywords are important
behind-the-scenes, without interfering with the instructor’s actions.<span style="mso-spacerun:yes">  </span>This allows the capability to learn to score
each question more accurately over time.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>As this is one
of our intelligent capabilities (which are usually significantly complex than
our other capabilities), here we provide a detailed discussion of the features
and requirements, including installation guideline.<o:p></o:p></span></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal" style="margin-bottom:6.0pt"><b style="mso-bidi-font-weight:
normal"><span style='font-size:16.0pt;font-family:"Eras Medium ITC";color:#FF9900'>1.<span style="mso-tab-count:1">     </span>Details<o:p></o:p></span></b></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>The Question and
Answering capability is based on the basic chat capability that comes with
Conference XP.<span style="mso-spacerun:yes">  </span>It uses the communication
protocols common to all capabilities to multicast messages to all participants
in the Venue.<span style="mso-spacerun:yes">  </span>These messages are strings
containing pertinent information such as the identifier who sent the message
bound between xml-style tags which we have defined.<span style="mso-spacerun:yes">  </span>This simplifies the process of parsing the
messages when they are received.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>The Question and
Answering capability is designed to run in either the I-MINDS or the CXP
Client.<span style="mso-spacerun:yes">  </span>There are actually two separate
versions of the capabilities which look the same to the CXP—the teacher and
student versions.<span style="mso-spacerun:yes">  </span>These versions differ
in many ways, the most significant being their capability forms and the teacher
version connecting to a server-side database.<span style="mso-spacerun:yes"> 
</span>The database is discussed in more detail below although it is worth
pointing out that some tables are updated by the I-MINDS Client.<span style="mso-spacerun:yes">  </span>This means that some tables which should be
updated are not when Q&amp;A is run using the CXP client.<span style="mso-spacerun:yes">  </span>These have minimal impact on the Q&amp;A
client and are summarized in table 1.1.<span style="mso-spacerun:yes"> 
</span>Screenshots for the Student and the Teacher version of Q&amp;A can be
found in figure 1.1 and 1.2 respectively<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:.5in"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<ul style="margin-top:0in" type="disc">
<ul style="margin-top:0in" type="circle">
<li class="MsoNormal" style="mso-list:l0 level2 lfo1;tab-stops:list 1.0in"><span style='font-family:"Eras Medium ITC"'>The first of these is that the
      sessionID for messages stored will not accurately reflect the session when
      the message was received.<span style="mso-spacerun:yes">  </span>Instead
      the most current session will be used or 0 if no sessions are found. <o:p></o:p></span></li>
<li class="MsoNormal" style="mso-list:l0 level2 lfo1;tab-stops:list 1.0in"><span style='font-family:"Eras Medium ITC"'>Because the student and teacher
      identifiers are not stored in Q&amp;A it is possible that a sender
      identifier for a message stored will not be found in either table. <o:p></o:p></span></li>
<li class="MsoNormal" style="mso-list:l0 level2 lfo1;tab-stops:list 1.0in"><span style='font-family:"Eras Medium ITC"'>The StudentsInSession table will
      not be populated at all so no record of which students join the classroom
      (Venue) will be kept.<o:p></o:p></span></li>
</ul>
</ul>

<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-size:16.0pt;font-family:"Eras Medium ITC";color:#FF9900'>2.<span style="mso-spacerun:yes">  </span><span style="mso-tab-count:1">   </span>Client<o:p></o:p></span></b></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>When designing
I-MINDS we desired that it have most of the same functionality found in the CXP
client.<span style="mso-spacerun:yes">  </span>First and foremost the user
should be able to see which Venues are available and be able to choose one to
connect to.<span style="mso-spacerun:yes">  </span>After connecting to the
Venue the user should have the ability to ability to both send and receive
multicast video and audio.<span style="mso-spacerun:yes">  </span>The user
should be able to update his or her profile.<span style="mso-spacerun:yes"> 
</span>Beyond this the capabilities that the user can start depend heavily on
whether the user is a student or teacher.<span style="mso-spacerun:yes"> 
</span><o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>The student has
very limited menu options; we expect the teacher to initiate most of the
capabilities.<span style="mso-spacerun:yes">  </span>The student can edit their
profile, services, and audio/video options.<span style="mso-spacerun:yes"> 
</span>The students can also record or playback conferences from any machine
(including the teachers).<span style="mso-spacerun:yes">  </span>In the future
we intend to limit student access to recorded conferences on the teacher’s
machine.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>The teacher has
the same options as the students to edit their profile, services, and
audio/video options.<span style="mso-spacerun:yes">  </span>They also have the
option to record or playback conferences from any machine.<span style="mso-spacerun:yes">  </span>In addition, the teacher’s Conference menu
contains options to start a Presentation capability from CXP.<span style="mso-spacerun:yes">  </span>Note that this is the CXP version not the UW
Classroom Presenter.<span style="mso-spacerun:yes">  </span>The teacher can
also startup one of three capabilities designed by the I-MINDS team—Group Chat,
Video Manager, or Video Manager Plus.<span style="mso-spacerun:yes"> 
</span>These are described in detail in their own documentation.<span style="mso-spacerun:yes">  </span>In the Agent menu the teacher has the option
to Start Q&amp;A.<span style="mso-spacerun:yes">  </span>This option starts the
Question Answering capability for everyone in the Venue.<span style="mso-spacerun:yes">  </span>The Add Keyword option brings up a form where
keywords and weights can be added or changed outside the Q&amp;A capability.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-size:16.0pt;font-family:"Eras Medium ITC";color:#FF9900'>3.<span style="mso-spacerun:yes">  </span><span style="mso-tab-count:1">   </span>Database<o:p></o:p></span></b></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>The I-MINDS
database is currently running MySQL version 4.1.<span style="mso-spacerun:yes">  </span>In the future we intend to switch over from
MySQL to SQL Server or something similar.<span style="mso-spacerun:yes"> 
</span>The basic structure should easily transfer over to any relational
database.<span style="mso-spacerun:yes">  </span>The database was designed
using a combination of DBDesigner4, an open source designer from fabForce, and
phpMyAdmin-2.6.3.<span style="mso-spacerun:yes">  </span>For an illustration of
the I-MINDS database consult figure 3.1 below.<span style="mso-spacerun:yes"> 
</span>The mysqldump textfile of the database can be found in the same folder
as the source code.<span style="mso-spacerun:yes">  </span><o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>On the C# side,
the database hook is contained in the ComponentTestLibrary.dll.<span style="mso-spacerun:yes">  </span>This dll, whose source is provided, consists
of a ComponentModel class called DatabaseWrapper and another class called
DatabaseMethods, along with the dataset schema DataSet1.xsd.<span style="mso-spacerun:yes">  </span>The DatabaseWrapper class consists of
numerous DataAdapters designed with the form editors.<span style="mso-spacerun:yes">  </span>The dataset schema was generated from these
DataAdapters using the wizard.<span style="mso-spacerun:yes">  </span><o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p align="center" class="MsoNormal" style="text-align:center"><span style='font-family:"Eras Medium ITC"'><!--[if gte vml 1]><v:shape id="_x0000_i1028"

<p align="center" class="MsoNormal" style="text-align:center"><b style="mso-bidi-font-weight:
normal"><span style='font-family:"Eras Medium ITC"'>Figure 3.1:</span></b><span style='font-family:"Eras Medium ITC"'><span style="mso-spacerun:yes"> 
</span>The DBDesigner model for the I-MINDS database<o:p></o:p></span></p>
<p align="center" class="MsoNormal" style="text-align:center"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>Wherever
possible we have tried to conform to the C# database connection model, filling
the dataset from multiple DataAdapters.<span style="mso-spacerun:yes"> 
</span>The tables in the dataset can then be hooked to DataGrids or DataViews
for display.<span style="mso-spacerun:yes">  </span>We use OdbcCommand builders
to generate both the update and insert queries with two exceptions.<span style="mso-spacerun:yes">  </span>We the Keyword_Ranking and Message
DataAdapters we experienced numerous DatabaseConcurrency exceptions which
claimed that 0 rows were updated.<span style="mso-spacerun:yes"> 
</span>Despite much experimentation we were unable to lock down this
exception.<span style="mso-spacerun:yes">  </span>Our solution was to write our
own Update and Insert queries and run them on each row in the DataTables based
on row states.<span style="mso-spacerun:yes">  </span>This seemed to eliminate
the problem.<o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><b style="mso-bidi-font-weight:normal"><span style='font-size:16.0pt;font-family:"Eras Medium ITC";color:#FF9900'>4.<span style="mso-spacerun:yes">  </span><span style="mso-tab-count:1">   </span>Installation<o:p></o:p></span></b></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>This section
leads you through the installation for ConferenceXP and our capabilities.<span style="mso-spacerun:yes">  </span>The first program to install is the
mysql-connector-odbc-3.51 driver.<span style="mso-spacerun:yes">  </span>We
have included this with our source code.<span style="mso-spacerun:yes"> 
</span>Next, if you have not already done so download the CXP Client version
3.2.<span style="mso-spacerun:yes">  </span>Before you install it make sure that
the Microsoft Journal Viewer has been installed.<span style="mso-spacerun:yes">  </span>Please consult the ConferenceXP client
installation guide for more information.<span style="mso-spacerun:yes"> 
</span><o:p></o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'><o:p> </o:p></span></p>
<p class="MsoNormal"><span style='font-family:"Eras Medium ITC"'>We provide the
database dump file so that users can upload the I-MINDS database to their local
server.<span style="mso-spacerun:yes">  </span>The installation is relatively
straight forward assuming you have access to MySQL server version 4.1 or
higher.<span style="mso-spacerun:yes">  </span>Table 4.1 gives the console
command.<span style="mso-spacerun:yes">  </span>If your machines do not have
access to the internet you will need to install your own venue service on one
of your machines.<span style="mso-spacerun:yes">  </span>The Venue Service can
be downloaded from the ConferenceXP website. <o:p></o:p></span></p>
